### Known Considerations

- **`App.initialize()`** — Currently not called from MainApplication.onCreate(). The sync engine
  initializes lazily when NativeSyncModule is first used. For production, consider adding
  `App.initialize(this)` to MainApplication.onCreate() for early initialization.

- **Billing integration** — BillingManager.kt is included and checks subscription status before
  allowing sync. It calls `GET /subscription` on the billing API. For development/testing, sync
  is allowed optimistically when the API is unreachable.

- **The `android/` directory** is gitignored by default in Expo projects. The sync engine source
  files inside it are regenerated by the config plugin (NativeSyncModule, NativeSyncPackage,
  XML resources, manifest entries) on each `expo prebuild`. However, the **copied Kotlin sync
  engine files** (48 files in `io/silentsuite/sync/`) are NOT regenerated by the plugin and
  must be re-copied after a clean prebuild.

- **Future improvement**: Move the sync engine copy step into the config plugin using
  `withDangerousMod` to copy files from `silentsuite-android/` automatically during prebuild.

## Architecture

```
React Native JS
  └── NativeSyncModule.ts (TypeScript interface with stubs)
        └── NativeSyncModule.kt (Kotlin native module, generated by config plugin)
              ├── triggerSync() → ContentResolver.requestSync()
              ├── getSyncStatus() → AccountManager query
              └── setBridgeMode() → AccountManager add/remove
                    └── Android Sync Framework
                          └── SyncAdapterService (Kotlin, from silentsuite-android)
                                └── SyncManager → Etebase SDK → E2EE sync
```

## Rebuilding After Changes

If you need to rebuild from scratch:

```bash
# 1. Regenerate android/ directory
export ANDROID_HOME=$HOME/Android/Sdk
npx expo prebuild --platform android --clean

# 2. Re-copy sync engine files
SRC="../../../../silentsuite-android/app/src/main/java/io/silentsuite/sync"
DEST="android/app/src/main/java/io/silentsuite/sync"
# (copy syncadapter/, resource/, model/, log/, utils/, billing/, root files)
# Then re-apply all refactoring patches (R import, UI dep removal, etc.)

# 3. Re-add submodule references and dependencies
# (These are in settings.gradle and app/build.gradle, which get overwritten by prebuild)

# 4. Build
cd android && ./gradlew :app:compileDebugKotlin
```
